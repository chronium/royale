Forward SDL events to ImGui while preserving game input and mouse-capture behavior.

## Completion Notes

Implemented UI-001 with Evergine ImGui.Net plus a project-owned `royale_imgui` native C ABI shim.

Native build:

* `thirdparty/fetch-imgui-net.sh` now fetches the pinned ImGui.Net checkout and required pinned submodules.
* `thirdparty/build-imgui-macos.sh` builds macOS ARM64 `thirdparty/artifacts/imgui/osx-arm64/lib/libroyale_imgui.dylib`.
* The dylib includes cimgui plus Dear ImGui's SDL3 platform backend and SDL_GPU renderer backend.
* The dylib uses SDL3 headers without linking a second SDL3 runtime library; the running client resolves SDL symbols through the SDL3-CS native library already copied beside `Royale.Client`.

Runtime behavior:

* `Royale.Client` references `thirdparty/repos/ImGui.Net/Generator/Evergine.Bindings.Imgui/Evergine.Bindings.Imgui.csproj`.
* macOS ARM64 client output copies `libroyale_imgui.dylib`.
* ImGui initializes after SDL window and SDL_GPU device creation.
* Every polled SDL event is forwarded to ImGui before game input handling.
* Each frame updates ImGui display size, framebuffer scale, and delta time.
* UI-001 begins and ends ImGui frames but does not submit draw data.
* ImGui backend/context state is destroyed before SDL_GPU and SDL window teardown.

Input policy:

* Quit, window close, and resize remain global.
* `F1` toggles relative mouse mode globally.
* `Escape` releases relative mouse mode before quitting.
* Relative mouse mode keeps game input active even if ImGui capture flags are set.
* With the mouse free, ImGui `WantCaptureKeyboard` and `WantCaptureMouse` suppress matching game input.
* Key/button releases for already-held game inputs are still applied to avoid stuck input state.

UI-002 boundary:

* UI-002 owns ImGui draw-data submission.
* Required SDL_GPU ordering is documented in `architecture/content-and-rendering`: render ImGui to produce draw data, call `ImGui_ImplSDLGPU3_PrepareDrawData()` before the ImGui render pass, then call `ImGui_ImplSDLGPU3_RenderDrawData()` inside that render pass.
