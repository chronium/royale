Implement or integrate an SDL GPU renderer backend for ImGui.

## Completion Notes

Implemented SDL_GPU draw-data submission through the existing UI-001 ImGui backend and native `royale_imgui` shim.

Frame ordering:

* SDL events are polled and forwarded to ImGui before game input handling.
* The ImGui frame starts after event polling.
* `SdlApplication` builds a minimal debug overlay before rendering.
* `SdlGpuDevice` acquires the command buffer and swapchain texture, renders the cube in the existing depth-enabled pass, then renders ImGui in a second color-only pass over the same swapchain texture with load `LOAD` and store `STORE`.
* `ImGui_ImplSDLGPU3_PrepareDrawData()` is called after `igRender()` and before the ImGui render pass begins.
* `ImGui_ImplSDLGPU3_RenderDrawData()` is called inside the ImGui render pass.
* Screenshot readback runs after ImGui rendering so captured frames include the overlay.

Overlay contents:

* Window title: `Royale`
* Frame delta and FPS
* Fixed ticks this frame
* Total fixed tick
* Mouse capture state

Added tests for invariant overlay state text formatting.
