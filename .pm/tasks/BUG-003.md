In a connected client, F7 DebugOnly renders the network player capsule, spawn markers, and safe-zone circle but omits every static collision wireframe, including the floor, perimeter walls, and interior boxes. F5 and F6 still render world solids, and F8 still renders collision solids.

Root cause: DebugSceneBuilder appends Box3D static-world lines only through localPlayer.CollisionWorld. Connected clients intentionally have localPlayer == null; their equivalent MapStaticCollisionWorld is already owned by ClientMovementPrediction, but it is not exposed to the debug rendering path.

Implement the smallest ownership-safe correction: let connected debug rendering consume the existing prediction collision world without creating a duplicate physics world or changing prediction, protocol, authority, map schema, or solid-rendering behavior. Preserve offline F6/F7 behavior and prediction-world disposal ownership.

Acceptance:
- Connected F6 includes static collision wireframes over world solids.
- Connected F7 includes floor, wall, and box wireframes alongside player/spawn/safe-zone primitives.
- Offline F6/F7 behavior remains unchanged.
- F5 and F8 routing remains unchanged.
- Automated coverage proves the connected debug builder receives static collision geometry and prevents regression.
- Human validation covers F5-F8 in a connected client.
