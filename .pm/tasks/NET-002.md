Define packet headers with protocol version, session identity, message type, sequencing, and acknowledgements.

## Completion Notes

- Added `Royale.Protocol` v1 packet framing with a fixed 29-byte little-endian header: magic `ROYL`, major/minor version, `SessionId`, `ProtocolMessageType`, `Sequence`, `AcknowledgedSequence`, and `AcknowledgementMask`.
- Added span-based header and packet read/write helpers. Packet payloads remain opaque to the framer and message body serialization is deferred.
- Defined initial message type values 1 through 8 for `ClientHello`, `ServerAccept`, `ServerReject`, `ClientInput`, `ServerSnapshot`, `ServerEvent`, `ClientDisconnect`, and `ServerDisconnect`.
- Documented `SessionId = 0` as pre-session only; accepted sessions use nonzero server-assigned ids. NET-003 will enforce handshake and session rules.
- Documented acknowledgement semantics: `AcknowledgedSequence` is the latest remote packet received; mask bit 0 acknowledges `ack - 1`; bit 31 acknowledges `ack - 32`, using wrapping `uint` arithmetic.
- Deferred transport behavior, handshake state machines, message body serialization, client/server launch wiring, input sending, and snapshot sending.
