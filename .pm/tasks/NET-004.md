Send the newest input plus recent commands to tolerate loss without reliable retransmission.

## Implementation Notes

- Added `ClientInputPayloadSerializer` in `Royale.Protocol` for 1 to 4 `PlayerInputCommand` records with a one-byte count and little-endian command fields.
- Added `ClientInputSender` in `Royale.Network` to frame `ProtocolMessageType.ClientInput` with the accepted nonzero session id and send newest-first redundant input via `NetworkDelivery.Unreliable` on channel `2`.
- Added `ServerInputReceiver` in `Royale.Network` to accept only accepted-peer input packets with the expected session id/channel, reject malformed payloads, filter duplicate or stale input by monotonic command sequence per peer, and forward accepted commands oldest-to-newest.
- Added protocol, network, and in-process handshake/input/session acknowledgement coverage. Executable SDL client input wiring, prediction/reconciliation, and dedicated-server loop wiring remain deferred.
- Updated `architecture/networking` with the input payload, delivery channel, redundancy window, stale filtering, and deferred wiring contract.
