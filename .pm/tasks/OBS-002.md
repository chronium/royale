Instrument connected-player counts, simulation tick timing, snapshot send rate, packet counts, invalid packets, input queue depth, connect/disconnect events, handshake failures, and match phase changes using low-cardinality metrics and structured logs.

## Implementation Notes

- Added `ServerObservability` in `Royale.Server` using `RoyaleTelemetry.ServerMeter` and server loggers.
- Wired `NetworkServerRuntime` to accept optional observability and leave non-instrumented construction as no-op.
- Kept telemetry ownership out of `Royale.Network`; network code only exposes observer callbacks for handshake rejects and invalid accepted-input packets.
- Exposed queued input count, living-player count, and match phase from `InProcessServerSession` for gauges.
- Metrics use low-cardinality labels only. Peer, connection, and player IDs are included in structured logs where useful, not in metric labels.
- Updated `architecture/observability` with the OBS-002 metric names and event policy.
