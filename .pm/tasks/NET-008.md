## Acceptance Notes

Implemented client-side reconciliation for accepted UDP clients:

- Authoritative local-player snapshots drop acknowledged pending inputs, seed prediction from the authoritative player state, and replay remaining unacknowledged commands in sequence order.
- Replay uses `PlayerMovementIntent.ToWorldMovement`, `KinematicCharacterController.Step`, and `SimulationSettings.FixedDeltaSeconds` against the client prediction collision world.
- Replayed commands are local-only and do not send network packets.
- `ClientNetworkState.LatestSnapshot` remains authoritative-only; reconciliation updates only predicted presentation state.
- Dead authoritative local-player snapshots resync prediction to the dead authoritative state and skip movement replay.
- `NetworkClientRuntime.Dispose()` now disposes prediction resources along with the transport.
- Added runtime diagnostics for last correction distance, last replayed input count, total reconciliation count, and existing pending input count.
- Remote-player interpolation, bad-network simulation, event replication, and cosmetic smoothing remain outside this task.

Updated `architecture/networking` to document implemented reconciliation behavior and remaining deferred networking work.
