Selecting a newly added navigation link crashes the editor because the viewport routes the non-spatial selection through EditorEntityTransforms.Get. Guard gizmo construction by transform capability and add regression coverage.

## Notes

- 2026-07-13 09:19 UTC - Fixed the native crash by guarding viewport gizmo construction with EditorEntityTransforms.HasSpatialTransform before reading the selected entity transform. Navigation links remain hierarchy-selectable but no longer enter the spatial transform path. Added regression coverage for non-spatial link routing. Validation: focused regression tests passed (8/8), full Royale.Editor.Tests passed (171/171), editor project build passed with 0 warnings/errors, scoped formatter verification passed, and git diff --check passed. The fixed native editor is running for owner retest. No wiki update was needed because architecture/editor already states that navigation links are non-spatial and never receive a gizmo; this fix restores that documented contract.
- 2026-07-13 09:19 UTC - Owner retested Add Navigation Link in the running native editor and confirmed it now works correctly without crashing.
