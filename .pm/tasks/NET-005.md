Transmit full authoritative snapshots at an initial target rate of 20 Hz from a 60 Hz simulation.

## Completion Notes

Implemented full `ServerSnapshot` payload serialization in `Royale.Protocol` with little-endian primitives, explicit nullable markers, UTF-8 length-prefixed weapon ids, and bounded payload constants (`MaxSnapshotPlayers = 128`, `MaxSnapshotWeaponIdLength = 64`). Deserialization rejects malformed, truncated, trailing, oversized, and invalid enum/marker payloads.

Added `ServerSnapshotSender` in `Royale.Network`. It consumes accepted peer/session metadata plus a per-peer snapshot provider, frames `ProtocolMessageType.ServerSnapshot` packets with accepted nonzero session ids, sends on `NetworkDelivery.Sequenced` channel `1`, and emits only every third 60 Hz server tick for the initial 20 Hz cadence.

Covered protocol round trips/rejections, sender cadence/framing/channel/session behavior, and an in-process handshake-to-snapshot integration path using `NetworkHandshakeServer.AcceptedPeers` and queued `InProcessServerSession` snapshots.

Updated `architecture/networking` to document full snapshot serialization, sender cadence/delivery/channel behavior, and deferred client receive/interpolation/executable wiring.
