Create Kustomize-managed local development manifests for Grafana, Prometheus, Loki, Tempo, and the OpenTelemetry Collector using separate Deployments and Services, with local customization kept out of committed secrets and machine-specific files.

## Notes

- Added `deploy/observability/base` with a `royale-observability` namespace, separate Deployments and Services for Grafana, Prometheus, Loki, Tempo, and the OpenTelemetry Collector, PVCs for Grafana/Prometheus/Loki/Tempo, and ConfigMaps generated by Kustomize.
- Extracted Prometheus, Loki, Tempo, and Collector application config into standalone files under `deploy/observability/base/config/`; service and deployment manifests no longer embed those YAML payloads directly.
- Added `deploy/observability/local` with an ignored `kustomization.yaml`, a copyable `kustomization.yaml.example`, copyable local config examples under `local/config/*.example`, and local apply/port-forward instructions.
- Local config definitions copied from the examples are ignored and can replace the base generated ConfigMaps through commented `configMapGenerator behavior: replace` blocks in the local kustomization.
- Pinned and verified registry tags: `grafana/grafana:13.1.0`, `prom/prometheus:v3.13.0`, `grafana/loki:3.7.3`, `grafana/tempo:2.10.7`, and `otel/opentelemetry-collector-contrib:0.156.0`.
- Collector accepts OTLP on `4317` and `4318`, exports Prometheus metrics on `:9464`, exports traces to Tempo over OTLP gRPC, and exports logs to Loki over OTLP HTTP at `/otlp`.
- Updated the Collector config to use the current `otlp_grpc` and `otlp_http` exporter type names instead of deprecated aliases reported by Collector `0.156.0`.
- Updated `architecture/observability` with the committed layout, generated ConfigMap policy, ports, PVC defaults, local overlay policy, and validation commands. Added a short README pointer. Grafana datasources and dashboards remain deferred to `OBS-005`.
