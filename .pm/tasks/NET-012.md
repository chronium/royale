Reduce visible local-player jitter after reconciliation, especially small jumps on movement key release and obvious jitter during diagonal movement. Preserve server authority and keep authoritative snapshots immutable.

## Acceptance Notes

Implemented a local-only presentation smoothing layer for predicted local-player corrections:

- Added `LocalPredictionSmoother` in the client presentation layer.
- The SDL render path now computes one smoothed predicted local player per frame and uses it for both gameplay camera and debug presentation snapshot.
- Small reconciliation corrections preserve the previous displayed position and decay a visual offset over presentation frames.
- Large corrections and dead local-player snapshots snap immediately.
- Smoothing does not mutate `ClientNetworkState.LatestSnapshot`, predicted simulation state, network packets, or server-owned gameplay state.

The observed server/client movement mismatch may still warrant future investigation at the input cadence or server input-hold policy level, but this task intentionally keeps gameplay authority unchanged and smooths residual hard corrections visually.

Updated `architecture/networking` to document local correction smoothing and remaining deferred interpolation/bad-network work.
