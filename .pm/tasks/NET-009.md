Add configurable latency, jitter, loss, duplication, and reordering for development testing. Use LiteNetLib capabilities where they fit, but expose bad-network controls through game-owned transport/test abstractions rather than directly from gameplay code.

## Implementation Notes

- Added `SimulatedNetworkTransport` as a game-owned `INetworkTransport` wrapper that delegates lifecycle operations to an inner transport and impairs only outbound `Send` packets and inbound `PacketReceived` events.
- Added `SimulatedNetworkConditions` with fixed latency, jitter, packet loss, duplication, reordering, and optional deterministic random seed support. `None` applies no impairment.
- Used `TimeProvider` injection so tests can advance simulated time without sleeps.
- Intentionally deferred client/server CLI flags, WattleScript scenario controls, debug UI toggles, and Grafana/observability actions to later tasks.
