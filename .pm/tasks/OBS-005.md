Provision Grafana datasources and starter dashboards for server health, networking, match state, logs, and traces using committed dashboard/config templates suitable for the local observability stack.

## Implementation Notes

- Added committed Grafana datasource provisioning for Prometheus, Loki, and Tempo with stable UIDs `royale-prometheus`, `royale-loki`, and `royale-tempo`.
- Added committed Grafana dashboard provider provisioning and three starter dashboards under the `Royale` folder:
  - `Royale Server Overview`
  - `Royale Networking`
  - `Royale Logs and Traces`
- Mounted the generated Grafana ConfigMaps into the Grafana Deployment at:
  - `/etc/grafana/provisioning/datasources`
  - `/etc/grafana/provisioning/dashboards`
  - `/var/lib/grafana/dashboards`
- Updated `deploy/observability/local/README.md` and the `architecture/observability` wiki page with provisioning details and local validation steps.

## Validation Notes

- Ran `kubectl kustomize deploy/observability/base` successfully.
- Ran `kubectl kustomize deploy/observability/local` successfully using the existing ignored local overlay.
- Ran `kubectl apply --dry-run=client -k deploy/observability/local` successfully.
- Ran `kubectl apply -k deploy/observability/local` successfully on the local OrbStack cluster.
- Ran `kubectl -n royale-observability rollout status deployment/grafana`; rollout completed successfully.
- Confirmed Grafana logs inserted datasources `royale-prometheus`, `royale-loki`, and `royale-tempo` and finished dashboard provisioning.
- Confirmed Grafana database state contains the three datasource UIDs and unified dashboard resources for `royale-server-overview`, `royale-networking`, and `royale-logs-traces` in the `royale` folder.
- Ran an OTLP-enabled finite server smoke. The first sandbox run hung with no output, matching the documented sandbox OTLP note; the elevated rerun with `OTEL_EXPORTER_OTLP_ENDPOINT=http://127.0.0.1:4317` completed successfully for 300 ticks.
- Confirmed Prometheus exposed observed series including `royale_server_players_connected`, `royale_server_players_active`, `royale_server_match_living_players`, `royale_server_match_phase`, `royale_server_inputs_queue_depth`, and `royale_server_tick_duration_milliseconds_*`.
- Confirmed Loki query `{service_name="royale-server"}` returned the server start/stop logs from the smoke.
- Confirmed Tempo query `{resource.service.name="royale-server"}` returned the `royale.server.run` trace from the smoke.
