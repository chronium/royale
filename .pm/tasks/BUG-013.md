Prevent valid map files with display names longer than the editor's fixed ImGui UTF-8 buffer from crashing during load. Define and enforce safe bounded editor input behavior without splitting UTF-8 sequences, and cover long ASCII and multibyte names.

## Notes

- 2026-07-13 08:33 UTC - Implemented a reusable 256-byte `EditorUtf8InputBuffer` with a 255-byte payload, complete-sequence UTF-8 truncation, null termination, truncation state, decoded value access, and stale-byte clearing. The Inspector now preserves over-limit authoritative names during load/undo/redo synchronization, tracks actual user edits before creating `SetMapNameCommand`, and shows the wrapped preservation warning in both Inspector and Validation. Added six focused tests covering short/exact/long ASCII/multibyte boundaries, strict UTF-8 validity, stale-byte clearing, and document preservation.
