Buffer and interpolate remote-player snapshots with position and rotation smoothing.

## Acceptance Notes

Implemented `RemoteSnapshotInterpolator` on the client with a bounded authoritative snapshot history and a default 6-server-tick / approximately 100 ms interpolation delay. `NetworkClientRuntime` now feeds the history from accepted snapshot packets and exposes diagnostics for buffered snapshot count, delay ticks, last target tick, and interpolation-vs-fallback use.

Presentation snapshots are built from the latest authoritative snapshot, interpolate remote-player position, velocity, yaw, and pitch when bracketing samples exist, and then substitute the smoothed predicted local player when available. `ClientNetworkState.LatestSnapshot` remains authoritative-only and is not mutated by prediction or interpolation.

Fallback behavior renders latest authoritative remote state until enough samples exist, uses the nearest buffered transform when a remote player is missing from a bracket, and holds the nearest buffered transform when the target tick is outside the buffer. No protocol, snapshot cadence, bad-network simulation, delta compression, event replication, or remote extrapolation changes were made.
