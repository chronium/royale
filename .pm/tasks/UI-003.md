Display real frame, simulation, player, physics, server, latency, packet, and connection statistics after the relevant gameplay, server, and networking telemetry sources exist.

## Implementation Notes

Implemented a default-visible, F3-toggled, resizable `Telemetry` ImGui window with collapsible Frame, Simulation, Player, Physics, Server, Network, and Connection sections. Hiding telemetry suppresses the separate Training Dummy window while leaving ImGui frame/render integration active. Offline mode omits Server and Network, retains Player kill/respawn and Training Dummy tooling, and connect mode uses explicit acceptance/snapshot waiting states.

Added game-owned optional transport diagnostics through `INetworkTransportDiagnostics` and immutable `NetworkPeerStatistics`. LiteNetLib statistics are enabled and projected without leaking LiteNetLib types; simulated transport forwards the optional interface. `NetworkClientRuntime` caches last-known transport statistics and owns input/snapshot/error/disconnect/latency/jitter diagnostics. Server, match, authoritative player, weapon/ammunition, and safe-zone fields remain sourced only from the latest accepted snapshot.

Updated `diagnostics`, `architecture/diagnostics-testing-deployment`, and `architecture/networking`.
